# BudgetFlow Roadmap

## Sprint Goal

Turn BudgetFlow into a credible public portfolio repo with steady daily GitHub activity for 30 days.

## Week 1: Foundation

- Initialize Git and prepare the public repo structure
- Clean tracked files and ignore generated leftovers
- Improve the README so the project is runnable and understandable fast
- Add license, contributing guide, roadmap, architecture notes, and GitHub templates
- Set up issues, milestones, and the daily work tracker

## Week 2: Product Polish

- Tighten auth copy and validation feedback
- Improve dashboard messaging and empty states
- Improve category and transaction form clarity
- Refine mobile responsiveness
- Capture initial screenshots for the README

## Week 3: Quality and Deployment

- Expand backend tests
- Add a manual QA checklist or lightweight frontend smoke coverage
- Improve API error handling in the frontend
- Add deployment notes for backend and frontend hosting
- Clarify production environment variables and deployment order

## Week 4: Portfolio Finish

- Publish release notes
- Improve repo visuals and social proof
- Add a "what I learned" narrative
- Fix the roughest UX or DX problems found during testing
- Tighten the README opening and final presentation

## After the 30-Day Sprint

- Add recurring transactions
- Add monthly budget-vs-actual insights
- Add CSV import/export
- Add real production deployment
- Add stronger frontend test coverage
