# BudgetFlow Sprint Issue Roadmap

This document is the public source of truth for BudgetFlow's 30-day GitHub sprint issue structure. Each day should close with one meaningful commit, one pushed change, and one visible repo improvement.

## Issue Workflow

1. Create the weekly milestones from `.github/milestones.yml`.
2. Sync the label catalog from `.github/labels.yml`.
3. Open one issue for each daily sprint task using the Sprint task issue form.
4. Assign each issue to its weekly milestone and suggested labels.
5. Close the daily issue only after the commit is pushed and the visible output is documented.

## Daily Closeout

Use this checklist in the closing comment for each sprint issue:

- What changed today?
- What is now visibly better in the repo?
- What issue was closed or updated?
- What is tomorrow's smallest meaningful improvement?
