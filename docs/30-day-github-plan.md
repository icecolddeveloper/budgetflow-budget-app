# 30-Day GitHub Presence Plan

## Operating Rules

- Public repo name: `budgetflow-budget-app`
- Default branch: `main`
- Commit format: `day X: <clear outcome>`
- One meaningful commit and push every day
- One visible output every day: code, docs, tests, screenshots, deployment, or release note

## Weekly Milestones

- Week 1: Foundation
- Week 2: Product Polish
- Week 3: Quality + Deployment
- Week 4: Portfolio Finish

## Daily Schedule

- [ ] Day 1: Initialize Git, prepare repo structure, clean `.gitignore`, and push the foundation commit
- [ ] Day 2: Remove or ignore packaging leftovers and tighten repo identity in the README
- [ ] Day 3: Verify backend setup and improve backend quick-start docs
- [ ] Day 4: Verify frontend setup and improve the full-stack run guide
- [ ] Day 5: Add license, contributing guide, and roadmap visibility
- [ ] Day 6: Create GitHub milestones, labels, and issue structure
- [ ] Day 7: Capture first README screenshots for core views
- [ ] Day 8: Improve auth UX copy and validation clarity
- [ ] Day 9: Improve dashboard messaging and empty/loading states
- [ ] Day 10: Improve category helper text and delete messaging
- [ ] Day 11: Improve transaction flow clarity and descriptions
- [ ] Day 12: Improve mobile responsiveness for auth and dashboard
- [ ] Day 13: Improve mobile responsiveness for categories and transactions
- [ ] Day 14: Publish architecture decision notes
- [ ] Day 15: Expand backend test coverage for registration and one transaction flow
- [ ] Day 16: Add backend tests for insufficient-balance and transfer validation
- [ ] Day 17: Add frontend smoke coverage or complete the manual QA pass
- [ ] Day 18: Improve frontend API error messaging
- [ ] Day 19: Add reviewer walkthrough and sample test flow to the README
- [ ] Day 20: Improve repo visuals with screenshots or a banner image
- [ ] Day 21: Publish `v0.1.0` release notes
- [ ] Day 22: Add backend deployment prep docs
- [ ] Day 23: Add frontend deployment prep docs
- [ ] Day 24: Document production environment setup
- [ ] Day 25: Do a security and cleanup pass
- [ ] Day 26: Add social-proof sections and lessons learned
- [ ] Day 27: Fix the roughest UX or DX issue found during testing
- [ ] Day 28: Run final QA and fix the most obvious breakages
- [ ] Day 29: Publish `v0.2.0` or `v1.0.0-beta` release notes
- [ ] Day 30: Final portfolio polish and close the sprint

## End-of-Day Template

Use this every day:

- What changed today?
- What is now visibly better in the repo?
- What issue was closed or updated?
- What is tomorrow's smallest meaningful improvement?
