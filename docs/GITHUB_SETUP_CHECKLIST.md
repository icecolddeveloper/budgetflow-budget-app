# GitHub Setup Checklist

Use this checklist when turning the local project into a visible public GitHub repo.

## Repository

- [ ] Create a new public repo named `budgetflow-budget-app`
- [ ] Keep the local folder name `BudgetApp 2026`
- [ ] Default branch should be `main`
- [ ] Add the repo description: `Modern Django + React budget app with a documented 30-day public build sprint`
- [ ] Pin the repo on your GitHub profile after the first push

## Labels to create

- [ ] `setup`
- [ ] `backend`
- [ ] `frontend`
- [ ] `ui`
- [ ] `docs`
- [ ] `tests`
- [ ] `deployment`
- [ ] `polish`

## Milestones to create

- [ ] `Week 1 Foundation`
- [ ] `Week 2 Product Polish`
- [ ] `Week 3 Quality + Deployment`
- [ ] `Week 4 Portfolio Finish`

## Issues to create

- [ ] Create 30 daily issues from `docs/github-issue-seeds.md`
- [ ] Assign each issue to the right weekly milestone
- [ ] Apply at least one label to every issue

## Releases to plan

- [ ] `v0.1.0` after documentation and repo foundation feel solid
- [ ] `v0.2.0` or `v1.0.0-beta` after polish, QA, and deployment prep

## Visibility checklist

- [ ] Add screenshots to the README
- [ ] Keep changelog entries current
- [ ] Push one meaningful improvement daily
- [ ] Close issues as work is completed
- [ ] Keep commit messages specific and readable
